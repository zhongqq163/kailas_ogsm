# 需求

更新时间：2026-06-22

## 背景

- 部门每个人都有 OGSM，目前主要沉淀在 Excel 中，维护、进度更新和内容调整不够方便。
- 现有样例 Excel：`IT数据应用支持组_钟启庆_OGSM_2026V2.0.xlsx`。
- 需要一个网页版本，支持随时查看、维护、修改内容和更新进度。

## 目标

- 建立可用的 AGSM 网页管理台。
- 将 Excel 中的 Object、Goal、Strategy、Measure、Tactics、主导人、完成日期、进度描述、协助人转成可编辑数据。
- 支持目标筛选、进度维护、状态维护、导入导出和后续多人协作扩展。

## 范围

- 第一版包含：纯前端页面、Excel 初始数据、本地浏览器保存、账号权限原型、领导看板、个人展示、维护中心、目标新增/编辑/删除、状态/负责人筛选、搜索、JSON 导入导出。
- 第一版不包含：登录权限、多人同时编辑、数据库、线上发布、外部系统集成。
- 后续可扩展：后端 API、数据库、账号权限、部门/人员维度、Excel 导入、操作日志、看板统计。

## 验收标准

- 打开 `web/index.html` 后可看到 Excel 初始化的 4 条 OGSM 目标。
- 领导默认进入看板页，可查看整体进度、成员进度、风险项和目标明细。
- 个人展示页可按人查看其 OGSM 展示页面。
- 维护中心作为独立入口，集中处理新增、修改、进度和导入导出。
- 管理员账号可查看全员数据，不显示维护中心入口；个人账号只查看和维护本人 OGSM。
- 展示页参考 OGSM 汇报排版：Object、统计卡、GOAL 标题条、策略条、指标/行动计划/进度/负责人表格。
- 每条行动计划可以维护是否完成，目标进度按 `已完成行动计划数 / 总行动计划数` 自动计算。
- 维护中心支持通过“新增行动项”输入框追加计划项；计划项同步写入 Tactics。
- 普通账号个人展示不显示成员选择栏，直接展示本人 OGSM。
- 当前账号权限为前端原型，只用于验证流程；正式上线必须由后端登录、接口权限和数据库共同控制。
- 可以编辑 OGSM 内容、负责人、协助人、完成日期、状态、进度和进度描述。
- 修改内容会自动保存在浏览器本地，刷新后仍保留。
- 可以按关键字、状态、负责人筛选。
- 可以导出 JSON 备份，也可以导入符合格式的 JSON。
